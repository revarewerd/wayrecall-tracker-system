### Задача 2.2: Connection Manager — дебаг логирование парсинга

**Проблема:** "пакет не парсится" — неясно почему и как выглядит

- [ ] Добавить DEBUG уровень логирования в `TeltonikaParser`, `WialonParser`, `RuptelaParser`, `NavtelecomParser`:

```scala
// ДО парсинга (в байтах/hex):
ZIO.logDebug(
  s"""
  |=== RAW PACKET RECEIVED ===
  |Protocol: $protocol
  |From: $remoteAddress
  |IMEI: ${buffer.slice(imeiStart, imeiEnd).toString}
  |Hex dump:
  |${HexFormat.of().formatHex(buffer.array())}
  |Length: ${buffer.readableBytes()} bytes
  """.stripMargin
)

// Успешный парсинг:
ZIO.logDebug(s"✓ Parsed $protocol: ${result.size} points")

// Ошибка парсинга:
ZIO.logError(
  s"""
  |✗ PARSE ERROR - $protocol
  |Expected format: ...
  |Got: ${HexFormat.of().formatHex(buffer.array())}
  |Error: ${error.message}
  """.stripMargin
)
```

- [ ] Создать конфиг `cm.debug.logRawPackets = true/false` (по умолчанию `false)
- [ ] Создать конфиг `cm.debug.logParseErrors = true` (по умолчанию `true`)
- [ ] Логи писать в отдельный файл `logs/cm-packets.log` для удобства анализа
