## Обнаруженные проблемы HW (анализ от 09.03)

### Критические расхождения Flyway V1 vs реальная БД:

| Аспект | Flyway V1 | Реальная БД | Код (INSERT) |
|--------|-----------|-------------|--------------|
| Колонка времени | `timestamp` | `time` | `time` |
| `organization_id` | **НЕТ** | ✅ ЕСТЬ | ✅ пишет |
| `is_moving` | **НЕТ** | ✅ ЕСТЬ | ✅ пишет |
| `is_valid` | **НЕТ** | ✅ ЕСТЬ | ✅ пишет |
| `protocol` | **НЕТ** | ✅ ЕСТЬ | ✅ пишет |
| `sensors` (jsonb) | **НЕТ** | ✅ ЕСТЬ | НЕ пишет |
| `location` (PostGIS) | **НЕТ** | ✅ ЕСТЬ | НЕ пишет |
| `ignition` | ✅ ЕСТЬ | **НЕТ** | НЕ пишет |
| `server_time` | ✅ ЕСТЬ | **НЕТ** | НЕ пишет |
| `inserted_at` | ✅ ЕСТЬ | **НЕТ** | НЕ пишет |

### Нейминг "telemetry" → должен быть "gps":
- `TelemetryPoint` → `GpsPosition`
- `TelemetryConsumer` → `GpsEventsConsumer`
- `TelemetryRepository` → `GpsPositionRepository`
- `TelemetryBatch` → `GpsPositionBatch`
- Комментарии: "ClickHouse" → "TimescaleDB"

### Прочее:
- Health check возвращает `ComponentCheck("clickhouse", "UP")` — должен быть `timescaledb`
- Документация описывает `gps_points` с UUID — реально `gps_positions` с BIGINT
- Порт в docs: 8091 — реально 10091
