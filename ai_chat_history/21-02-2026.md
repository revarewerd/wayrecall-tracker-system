GitHub Copilot: **Ref** - это ZIO абстракция для изменяемого состояния в функциональном стиле. 🎯

## Что такое Ref?

```scala
// Классическая мутация (imperative):
var lastPosition: Option[GpsPoint] = None
lastPosition = Some(newPoint)  // ❌ мутация, не safe

// ZIO Ref (functional):
val stateRef: Ref[ConnectionState] = ???
stateRef.update(_.copy(lastPosition = Some(newPoint)))  // ✅ safe, composable
```

## Как создается:

```scala
stateRef: Ref[ConnectionState] = 
  Unsafe.unsafe(implicit u => 
    runtime.unsafe.run(Ref.make(ConnectionState())).getOrThrow()
  )
```

В коде это в ConnectionHandler:
```scala
class ConnectionHandler(...) extends ChannelInboundHandlerAdapter:
  private val stateRef: Ref[ConnectionState] = 
    Unsafe.unsafe(implicit u => 
      runtime.unsafe.run(Ref.make(ConnectionState())).getOrThrow()
    )
```

## Операции с Ref:

```scala
// Прочитать значение
state <- stateRef.get
// Result: ConnectionState

// Изменить значение
_ <- stateRef.update(_.copy(lastPosition = Some(point)))
// Атомарно! Если 2 потока обновляют одновременно - ZIO разберётся

// Получить и изменить
newState <- stateRef.modify { state =>
  val newState = state.copy(...)
  (newState, result)  // (новое значение, что вернуть)
}
```

## Зачем нужен Ref вместо обычной переменной?

1. **Thread-safe** - несколько Netty потоков могут обновлять одновременно
2. **Composable** - работает с ZIO эффектами
3. **Immutable-friendly** - не переписываем переменную, создаём новое значение через `.copy()`
4. **Fast** - это просто обёртка вокруг `AtomicReference` под капотом

```scala
// Под капотом:
Ref[ConnectionState] ≈ AtomicReference[ConnectionState]
```

## В нашем контексте:

```scala

_ <- stateRef.update(_.copy(lastPosition = Some(point)))

// Это очень быстро (nanoseconds), потому что:
// 1. Работает в памяти (не Redis)
// 2. AtomicReference.compareAndSet под капотом
// 3. Нет блокировок, только CAS (Compare-And-Swap)
```

**TL;DR:** Ref = безопасная мутабельная переменная в ZIO, работает в памяти, очень быстра. ✅

## 🔑 Ключевой момент:

**ConnectionState хранит timestamp кэша:**
```scala
case class ConnectionState(
    ...
    cachedContext: Option[DeviceContext] = None,
    contextCachedAt: Long = 0L          // ← когда последний раз обновили
    contextCacheTtlMs: Long = 3600000   // ← 1 час
)
```

**На каждый GPS пакет проверяем:**
```scala
val isContextFresh = (now - contextCachedAt) < contextCacheTtlMs

if isContextFresh then
  use cachedContext  // ← В памяти, no Redis
else
  freshContext <- redisClient.getDeviceContext(imei)  // ← Fresh HGETALL
  update contextCachedAt = now
  use freshContext
```

```scala
_ <- redisClient.subscribe(s"device-config-changed:$imei") { _ =>

}
```

**Kafka device-events:**
- ✅ Гарантированная доставка (если CM был offline)
- ✅ История изменений
- ✅ Другие сервисы могут слушать (например, History Writer)

**Redis Pub/Sub:**
- ✅ Instant notification (как только пришло)
- ✅ Множество subscribers одновременно
- ✅ Intra-process быстро (в памяти)

```
PostgreSQL ← Device Manager
    ↓
Kafka device-events ← DeviceEventConsumer читает отсюда
    ↓
Redis device:{imei} ← DeviceEventConsumer пишет сюда
```

### 3. ✅ ДИНАМИЧЕСКИЙ КОНФИГ ФИЛЬТРОВ - УЖЕ ЕСТЬ И РАБОТАЕТ! 🚀

**Файл:** `src/main/scala/com/wayrecall/tracker/config/DynamicConfigService.scala` (148 строк)

**Как работает:**

```
1️⃣ ИНИЦИАЛИЗАЦИЯ (при старте CM)
   application.conf (defaults)
        ↓
   Redis HSET config:filters (сохраняем)
        ↓
   configRef: Ref[FilterConfig] (in-memory, ~10ns)

2️⃣ ИСПОЛЬЗОВАНИЕ (на каждый GPS пакет)
   DeadReckoningFilter.validate(point)
        ↓
   configService.getFilterConfig()  ← Ref.get (⚡ ~10 nanoseconds!)
        ↓
   применяем фильтр с текущей конфигурацией

3️⃣ ОБНОВЛЕНИЕ (через HTTP API)
   PUT /config/filters (new values)
        ↓
   Redis HSET config:filters (persisted)
        ↓
   PUBLISH config:updates (Pub/Sub notification)
        ↓
   Все инстансы CM получают:
   - configRef.set(newConfig)
   - используют новые значения НА СЛЕДУЮЩЕМ пакете

4️⃣ СИНХРОНИЗАЦИЯ (между инстансами)
   Redis Pub/Sub канал: config:updates
        ↓
   configRef.set(newConfig)  ← in-memory update
        ↓
   ВСЕ инстансы синхронизированы! ✅
```

**FilterConfig поля:**
