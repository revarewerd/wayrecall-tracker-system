# 📝 Система заметок в Markdown-документации

## Зачем?

При изучении проекта нужен способ оставлять заметки прямо в документации:
- Что непонятно и требует разъяснения
- Что вызывает сомнения в реализации
- Идеи для улучшения
- Места, где нужна помощь AI

## Теги заметок

Заметки пишутся как **HTML-комментарии** — они не видны при рендеринге Markdown,
но легко находятся поиском по файлам.

### Доступные теги

| Тег | Назначение | Пример |
|-----|-----------|--------|
| `<!-- ❓ ВОПРОС: ... -->` | Непонятное место, нужно объяснение | Как работает Haversine? |
| `<!-- ⚠️ СОМНЕНИЕ: ... -->` | Подозрение на баг или плохое решение | Правильный ли тут TTL? |
| `<!-- 💡 ИДЕЯ: ... -->` | Предложение по улучшению | Можно добавить retry |
| `<!-- 📌 TODO: ... -->` | Задача для реализации | Написать интеграционный тест |
| `<!-- 🔍 ИЗУЧИТЬ: ... -->` | Тема для глубокого погружения | Разобраться с ZIO Layers |

### Формат заметки

```markdown
<!-- ❓ ВОПРОС: [краткое описание]
Подробное описание что именно непонятно.
Контекст: в каком файле/методе встретил.
-->
```

### Примеры использования

**В документации сервиса:**
```markdown
## Архитектура ConnectionHandler

Основной обработчик TCP пакетов...

<!-- ❓ ВОПРОС: Почему ConnectionHandler использует @volatile var для detectedParser?
Не лучше ли было бы использовать ZIO Ref для потокобезопасности?
Контекст: services/connection-manager/src/main/scala/.../network/ConnectionHandler.scala
-->
```

**В коде (Scala комментарии):**
```scala
// ❓ ВОПРОС: Зачем тут двойной retry — сначала в writeBatchWithRetry, потом в consumeStream?
// ⚠️ СОМНЕНИЕ: batchConfig.maxRetries может быть 0, тогда retry не сработает
```

**В daily tasks:**
```markdown
## Задачи
- [ ] Разобраться с filter pipeline — ❓ непонятна последовательность DeadReckoning → Stationary
```

## Как AI обрабатывает заметки

При запросе **"разбери мои заметки"** или **"ответь на вопросы"** AI:

1. Ищет все файлы с тегами `❓ ВОПРОС`, `⚠️ СОМНЕНИЕ`, `💡 ИДЕЯ`, `🔍 ИЗУЧИТЬ`
2. Группирует по сервисам
3. Отвечает на каждый вопрос со ссылками на код
4. Предлагает заменить тег на `<!-- ✅ ОТВЕЧЕНО: ... -->`

### Команды для AI

| Команда | Действие |
|---------|----------|
| `разбери мои заметки` | Найти и ответить на все ❓ ВОПРОС |
| `покажи сомнения` | Найти все ⚠️ СОМНЕНИЕ и проанализировать |
| `покажи идеи` | Собрать все 💡 ИДЕЯ для обсуждения |
| `что осталось изучить` | Найти все 🔍 ИЗУЧИТЬ |
| `статус заметок` | Сводка: сколько вопросов, сомнений, идей |

## Поиск заметок

```bash
# Все вопросы в проекте
grep -rn "❓ ВОПРОС" docs/ services/

# Все сомнения
grep -rn "⚠️ СОМНЕНИЕ" docs/ services/

# Все заметки (любой тег)
grep -rn "ВОПРОС\|СОМНЕНИЕ\|ИДЕЯ\|ИЗУЧИТЬ" docs/ services/

# Отвеченные вопросы
grep -rn "✅ ОТВЕЧЕНО" docs/ services/
```

## Правила

1. **Язык** — русский (как вся документация)
2. **Формат** — HTML-комментарий `<!-- тег: текст -->`
3. **Контекст** — указывай файл/метод где встретил непонятное
4. **Краткость** — описание 1-3 предложения
5. **Не удалять** — после ответа AI менять тег на `✅ ОТВЕЧЕНО`
