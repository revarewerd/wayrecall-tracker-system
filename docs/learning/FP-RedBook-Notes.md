## Глава 2. Продолжение: main, Unit, запуск, REPL, модули

### Перевод книги

> В Scala значение многострочного блока в фигурных скобках — это результат последнего выражения в блоке. Поэтому результат метода formatAbs — это просто значение, возвращаемое вызовом msg.format(x, abs(x)).
>
> Наконец, наш метод main — это внешняя оболочка, которая вызывает наш чисто функциональный "ядро" и печатает ответ в консоль. Такие методы иногда называют процедурами (или нечистыми функциями), чтобы подчеркнуть, что они имеют побочные эффекты:
>
> ```scala
> def main(args: Array[String]): Unit =
>   println(formatAbs(-42))
> ```
>
> Имя main — особое: при запуске программы Scala ищет метод с этим именем и определённой сигнатурой. Он должен принимать Array[String] и возвращать Unit.
>
> Массив args содержит аргументы командной строки, с которыми запущена программа. Здесь мы их не используем.
>
> Unit выполняет ту же роль, что void в C/Java. В Scala любая функция должна что-то возвращать (если не упала с ошибкой). Но main не возвращает ничего осмысленного, поэтому есть специальный тип Unit, у которого только одно значение — (). Обычно возвращаемый тип Unit намекает, что функция имеет побочный эффект.
>
> Тело main печатает в консоль строку, возвращённую formatAbs. Обратите внимание, что println возвращает Unit — это как раз то, что нужно main.
>
> ---
>
> **2.2 Запуск программы**
>
> Самый простой способ запустить Scala-программу — скомпилировать её с помощью scalac:
>
>     scalac MyModule.scala
>
> Это создаст .class-файлы, которые можно запустить через JVM:
>
>     scala MyModule
>
> Либо можно сразу передать исходник интерпретатору:
>
>     scala MyModule.scala
>
> Это удобно для скриптов. Интерпретатор найдёт object с main и вызовет его.
>
> Ещё один способ — REPL (интерактивная оболочка):
>
>     scala
>     :load MyModule.scala
>     MyModule.abs(-42)
>
> REPL позволяет быстро пробовать выражения, видеть их результат и тип.
>
> ---
>
> **2.3 Модули, объекты и пространства имён**
>
> Чтобы обратиться к методу abs, нужно писать MyModule.abs, потому что abs определён внутри объекта MyModule — это его пространство имён (namespace). В Scala почти всё — объект, и у каждого объекта могут быть члены (методы, значения, другие объекты). Объект, основной задачей которого является предоставление пространства имён для своих членов, иногда называют модулем.
>
> Доступ к членам объектов осуществляется через точку: имя объекта, точка, имя члена.

### Пояснения

- **main** — точка входа, всегда принимает Array[String] и возвращает Unit. Если сигнатура не совпадает — программа не запустится как standalone.
- **Unit** — специальный тип с единственным значением (). Используется для функций с побочными эффектами (side effects), например, println, запись в файл, отправка в сеть.
- **REPL** — мощный инструмент для экспериментов: можно быстро проверить выражение, функцию, тип, не создавая отдельный файл.
- **Модули и пространства имён** — object в Scala часто используется как модуль: контейнер для функций, констант, фабрик. Это аналог static-класса в Java, но с более строгой иммутабельностью.
- **Доступ через точку** — стандартный способ обращения к членам объекта: MyModule.abs, DeviceManagerApp.main и т.д.

### Пример из Wayrecall Tracker

**main и Unit:**
```scala
object HistoryWriterApp {
  def main(args: Array[String]): Unit =
    ZIOAppDefault.run(args)
}
```
Здесь main — точка входа, возвращает Unit, запускает ZIO-приложение.

**Модуль и пространство имён:**
```scala
object RedisKeys {
  val DeviceCache = "device:%d"
  def deviceStatusKey(id: Long): String = s"device:${id}:status"
}

// Использование:
val key = RedisKeys.deviceStatusKey(42)
```
Здесь RedisKeys — модуль, который группирует все ключи для Redis. Доступ к функциям и значениям — только через имя объекта.

## Глава 1. Введение в функциональное программирование

### Перевод книги

> Функциональное программирование — это стиль программирования, в котором вычисления строятся как вычисление значений с помощью чистых функций, без побочных эффектов и мутаций.

### Пояснения

- Чистая функция: всегда возвращает один и тот же результат для одних и тех же аргументов, не изменяет внешнее состояние.
- Побочные эффекты (side effects): изменение состояния вне функции (запись в файл, изменение переменной, вывод в консоль и т.д.).
- Референциальная прозрачность: выражение можно заменить его значением без изменения поведения программы.

### Пример из Wayrecall Tracker

```scala
def add(a: Int, b: Int): Int = a + b // чистая функция
```

## Глава 2. Знакомство с языком Scala

### Перевод книги

> Мы еще не раз вернемся к этим концепциям с разных сторон на протяжении всей книги, а сейчас наша задача — просто дать вам первое представление.
>
> **Введение в язык Scala: пример**
>
> Ниже приведен полный листинг программы на Scala, который мы подробно разберем. Здесь не вводится никаких новых концепций функционального программирования — наша цель лишь познакомить вас с языком Scala и его синтаксисом.
>
> **Листинг 2.1. Простая программа на Scala**
>
> ```scala
> // Комментарий!
> /* Еще один комментарий */
> /** Документационный комментарий */
> object MyModule {
>   def abs(n: Int): Int =
>     if (n < 0) -n
>     else n
>
>   private def formatAbs(x: Int) = {
>     val msg = "The absolute value of %d is %d"
>     msg.format(x, abs(x))
>   }
>
>   def main(args: Array[String]): Unit =
>     println(formatAbs(-42))
> }
> ```
>
> Мы объявляем объект (также называемый модулем) с именем MyModule. Это просто место для нашего кода и имя, чтобы мы могли ссылаться на него позже. Код на Scala должен находиться либо в объекте, либо в классе, и мы используем объект, потому что это проще. Мы помещаем код внутрь объекта, между фигурными скобками. О объектах и классах мы поговорим подробнее чуть позже. Сейчас рассмотрим этот конкретный объект.
>
> В объекте MyModule три метода, объявленных с помощью ключевого слова def: abs, formatAbs и main. Мы будем использовать термин "метод" для обозначения любой функции или поля, определенного внутри объекта или класса с помощью def. Давайте теперь рассмотрим методы MyModule по очереди.
>
> **Ключевое слово object**
>
> Ключевое слово object создает новый тип-одиночку (singleton type), который похож на класс, но имеет только один именованный экземпляр. Если вы знакомы с Java, объявление объекта в Scala похоже на создание нового экземпляра анонимного класса.
>
> В Scala нет аналога ключевого слова static из Java, и объект часто используется там, где в Java вы бы использовали класс со статическими членами.
>
> **Метод abs — чистая функция, принимающая целое число и возвращающая его абсолютное значение:**
>
> ```scala
> def abs(n: Int): Int =
>   if (n < 0) -n
>   else n
> ```
>
> Ключевое слово def сопровождается именем метода, за которым следует список параметров в скобках. В данном случае abs принимает только один аргумент n типа Int. После закрывающей скобки списка аргументов идет необязательная аннотация типа (": Int"), указывающая, что результат имеет тип Int (двоеточие читается как "имеет тип").
>
> Тело метода идет после знака равенства (=). Мы иногда будем называть часть объявления до знака равенства "левая часть" или "сигнатура", а код после знака равенства — "правая часть" или "определение". Обратите внимание на отсутствие явного ключевого слова return. Возвращаемое значение метода — это просто результат вычисления правой части. Все выражения, включая if, возвращают результат. Здесь правая часть — это одно выражение, значение которого либо -n, либо n, в зависимости от условия n < 0.
>
> **Метод formatAbs — еще одна чистая функция:**
>
> ```scala
> private def formatAbs(x: Int) = {
>   val msg = "The absolute value of %d is %d."
>   msg.format(x, abs(x))
> }
> ```
>
> Здесь мы вызываем метод format у объекта msg, передавая ему значения x и abs(x). В результате получается новая строка, в которой %d заменяются на значения x и abs(x) соответственно.
>
> Этот метод объявлен как private, то есть его нельзя вызвать из кода вне объекта MyModule. Эта функция принимает Int и возвращает String, но обратите внимание, что возвращаемый тип не объявлен. Scala обычно может вывести тип возвращаемого значения, поэтому его можно опустить, но считается хорошим стилем явно указывать типы возвращаемых значений для методов, которые вы ожидаете, что будут использовать другие. Этот метод приватный, поэтому мы можем опустить аннотацию типа.
>
> Тело метода содержит более одного выражения, поэтому мы заключаем их в фигурные скобки. Пара скобок с выражениями называется блоком. Выражения разделяются переводом строки или точкой с запятой. В данном случае мы используем перевод строки, поэтому точка с запятой не нужна.
>
> Первая строка в блоке объявляет строку msg с помощью val. Это просто имя для строкового значения, чтобы мы могли ссылаться на него далее. val — это неизменяемая переменная, поэтому внутри formatAbs имя msg всегда будет ссылаться на одно и то же строковое значение. Компилятор Scala выдаст ошибку, если вы попытаетесь присвоить msg другое значение в том же блоке.
>
> Помните, что метод просто возвращает значение своей правой части, поэтому ключевое слово return не требуется. В данном случае правая часть — это блок.

### Пояснения

- **object** — это "модуль" или синглтон, аналог статических классов в Java. В вашем проекте, например, `object Main` или `object DeviceManagerApp` — это точки входа сервисов.
- **def abs(n: Int): Int** — объявление функции с явной сигнатурой. В Scala всегда указывается тип параметра и (желательно) тип возвращаемого значения.
- **val** — неизменяемая переменная (аналог `final` в Java, но короче и удобнее). В проекте Wayrecall Tracker все параметры конфигов, константы, промежуточные значения объявляются через val.
- **private def** — приватный метод, виден только внутри объекта/класса. В сервисах Wayrecall Tracker приватные методы часто используются для инкапсуляции вспомогательной логики (например, парсинг, валидация).
- **Scala не требует return** — результатом функции становится последнее выражение в блоке.
- **Блоки** — если тело функции состоит из нескольких выражений, они заключаются в фигурные скобки.

### Пример из Wayrecall Tracker

```scala
object DeviceManagerApp {
  def main(args: Array[String]): Unit =
    ZIOAppDefault.run(args)

  private def parseConfig(path: String): Task[AppConfig] =
    for {
      raw <- ZIO.attemptBlockingIO(Files.readString(Paths.get(path)))
      config <- ZIO.fromEither(decode[AppConfig](raw))
    } yield config
}
```

- Здесь `object DeviceManagerApp` — точка входа.
- `main` — публичный метод, запускающий приложение.
- `private def parseConfig` — приватная функция для парсинга конфига, возвращает эффект ZIO (аналог Task).
- Все переменные объявлены через `val` внутри for-комбинатора (иммутабельность).
- Нет ни одного `var` или явного `return`.

### Вопросы по object/class

- Нарушает ли object функциональное программирование?
- Какой тип у object в Scala?
- Как работают object и class с одинаковым именем (компаньоны)?

---

### Ответы на вопросы

**Object не нарушает ФП, если внутри нет мутаций и side effects.** Это просто контейнер для функций и констант. Тип object — уникальный singleton-тип, который нельзя создать вручную. В FP-проектах object часто используется как модуль с чистыми функциями.

**Класс и object с одним именем — это разные сущности, не конфликтуют.** Object — синглтон, класс — шаблон для экземпляров. Они могут "дружить" и совместно реализовывать паттерны (например, фабрика, type class).

**Пример:**
```scala
class Device(val id: Long, val imei: String)
object Device {
	def fromJson(json: String): Device = // парсинг
}
```
