# 📝 Заметки по ревью документации

> **Инструкция:** при чтении каждого документа добавляй заметки в соответствующую секцию.
> AI понимает этот формат: уровень критичности + номер строки/секции + описание.

---

## Условные обозначения

| Иконка | Значение |
|--------|----------|
| 🔴 | **Критично** — ошибка, несоответствие, неактуальные данные |
| 🟡 | **Важно** — пропущена информация, нужно дополнить |
| 🟢 | **Мелочь** — опечатка, стиль, улучшение |
| ❓ | **Вопрос** — не уверен, нужно обсудить |
| 💡 | **Идея** — предложение по улучшению |
| 🗑️ | **Удалить** — устаревшее, дублирующее |

---

## Формат заметки

```
### [Иконка] [Краткое описание]
- **Где:** [имя файла] → [секция или номер строки]
- **Проблема:** [что не так]
- **Ожидание:** [как должно быть]
```

### Пример:

### 🔴 Неправильный порт CM
- **Где:** ARCHITECTURE_BLOCK1.md → "Порты сервисов"
- **Проблема:** Указан порт 8090 для Admin API
- **Ожидание:** Должен быть 10090 (актуальный)

### 🟡 Нет мета-блока
- **Где:** ARCHITECTURE_BLOCK2.md → начало файла
- **Проблема:** Нет тега, даты, версии
- **Ожидание:** Добавить `> Тег: АКТУАЛЬНО | Обновлён: YYYY-MM-DD | Версия: X.X`

### 💡 Добавить resizable sidebar
- **Где:** WEB_FRONTEND.md → "Компоненты и экраны"
- **Проблема:** Sidebar фиксированный, нельзя расширить список машин
- **Ожидание:** Описать resizable panel как в STELS (ExtJS splitter)
